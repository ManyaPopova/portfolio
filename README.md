# Проект DWH для банка

Проект по построению хранилища данных банка: спроектирована модель «снежинка», реализованы два ETL-потока (подключение продуктов и транзакции), настроен контроль качества данных и аудит загрузок.

Проект реализован в качестве выпускного задания курса по DWH от компании T1
## Файлы в репозитории

- **S2T.xlsx** — маппинг (Source-to-Target): описание соответствия полей из источников в целевые таблицы DWH, типы данных, первичные и внешние ключи, трансформации.

- **Кодовая реализация.docx** — ETL-скрипты на SQL: инкрементальная загрузка двух таблиц фактов (подключение продуктов и транзакции) с использованием MERGE, оконных функций, временных таблиц и аудита.

- **Метрики качества данных.xlsx** — 18 метрик качества: полнота, уникальность, ссылочная целостность, свежесть. Для каждой указан SQL-запрос, пороги хорошего качества и владелец.

- **Проект работы хранилища данных.docx** — DDL-скрипты и физическая оптимизация: партиционирование, распределение (DISTRIBUTED BY), колоночное сжатие, пример аналитического запроса.

- **Схема потоков.docx** — архитектура ETL: описание двух независимых потоков данных, DAG-задачи в Airflow, слои стейджинга, валидации и загрузки в DWH.
